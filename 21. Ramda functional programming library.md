
# 🧠 Ramda.js

---

## 💡 1. Ramda는 어떤 라이브러리인가요?

> ✅ Ramda는 **함수형 프로그래밍을 쉽게 하기 위해 만든 자바스크립트 라이브러리**입니다.

**Q. 함수형 프로그래밍이 뭐예요?**
👉 "데이터를 직접 변경하지 않고, 함수를 이용해 조합하며 결과를 만드는 방식"입니다.

```js
// 일반적인 코드
const numbers = [1, 2, 3];
const doubled = [];

for (let i = 0; i < numbers.length; i++) {
  doubled.push(numbers[i] * 2);
}
```

```js
// 함수형 스타일 (Ramda 스타일)
const doubled = R.map(x => x * 2, [1, 2, 3]);
```

## 🧩 2. 왜 Ramda가 필요할까요?

### Lodash처럼 반복 작업을 줄여주지만,

> Lodash는 `_.map(배열, 함수)` 방식이고
> Ramda는 `R.map(함수)(배열)` 방식입니다. ← 이게 포인트

Ramda는 함수형 스타일을 기본으로 하기 때문에 아래 특징이 있습니다:

| 특징          | 설명                                 |
| ----------- | ---------------------------------- |
| ✅ 순수함수 사용   | 같은 입력 → 항상 같은 출력                   |
| ✅ 데이터 변경 없음 | 기존 배열이나 객체를 절대 바꾸지 않음              |
| ✅ 함수 조합     | 여러 함수를 연결해서 사용 (`pipe`, `compose`) |
| ✅ 자동 커링     | 모든 함수는 `R.fn(a)(b)`처럼 나눠서 써도 됨     |

## ✨ 3. 아주 쉬운 Ramda 함수 예제들

### ✔️ `R.map()`: 배열의 모든 요소에 함수 적용

```js
const double = x => x * 2;

R.map(double)([1, 2, 3]); // [2, 4, 6]
```

<br>

### ✔️ `R.filter()`: 조건에 맞는 요소만 남기기

```js
const isEven = x => x % 2 === 0;

R.filter(isEven)([1, 2, 3, 4]); // [2, 4]
```

<br>

### ✔️ `R.prop()`: 객체에서 속성 꺼내기

```js
const user = { name: 'Alice', age: 30 };

R.prop('name')(user); // 'Alice'
```

<br>

### ✔️ `R.pluck()`: 객체 배열에서 특정 속성만 추출

```js
const users = [
  { name: 'Alice', age: 30 },
  { name: 'Bob', age: 28 }
];

R.pluck('name')(users); // ['Alice', 'Bob']
```

## 🔁 4. Ramda의 진짜 강점: `R.pipe()`

> 여러 함수를 순서대로 연결해서 실행하는 도구입니다.
> `파이프처럼 데이터를 흘려보낸다`는 의미입니다.

```js
const result = R.pipe(
  R.map(x => x + 1),
  R.filter(x => x % 2 === 0)
)([1, 2, 3, 4]);
// [2, 4]
```

👉 `1 → 2 → 홀수 제거 → 2, 4만 남음`

## 🛠️ 5. 실전 예제: 성인 이름만 뽑기

```js
const users = [
  { name: "Alice", age: 30 },
  { name: "Bob", age: 17 },
  { name: "Tom", age: 19 }
];

const getAdultNames = R.pipe(
  R.filter(user => user.age >= 18),     // 성인만 걸러내고
  R.map(user => user.name)              // 이름만 추출
);

console.log(getAdultNames(users)); // ["Alice", "Tom"]
```

## 🔍 6. Ramda가 Lodash보다 좋은 점?

| 항목        | Lodash           | Ramda                   |
| --------- | ---------------- | ----------------------- |
| 함수 순서     | `_.map(arr, fn)` | `R.map(fn)(arr)`        |
| 커링 지원     | 수동 (`_.curry`)   | 기본 자동 커링                |
| 순수 함수 스타일 | 선택적              | 기본 적용                   |
| 코드 조합     | 체이닝              | `pipe()` 또는 `compose()` |

---

## 🎁 보너스: Ramda는 어떻게 생각해야 할까?

* ✅ 자바스크립트 배열 메서드 `map`, `filter`, `reduce` → Ramda로 치환 가능
* ✅ 순서대로 처리하고 싶다 → `pipe`
* ✅ 커링이 어렵다면 걱정 X: 그냥 `R.fn(a)(b)` 라고 생각하세요
* ✅ 가독성이 중요하다면: Ramda = 깔끔한 데이터 흐름

## 📌 정리: Ramda는 어떤 개발자에게 추천?

| 대상                            | 추천 정도    |
| ----------------------------- | -------- |
| 배열과 객체를 많이 다루는 프런트엔드          | 🔥 매우 추천 |
| 데이터 흐름을 함수로 표현하고 싶은 백엔드       | ✅ 추천     |
| Lodash가 불편하고 체이닝이 난잡하다고 느낀 경우 | 🔥 강력 추천 |
