자바스크립트에서 **객체(Object)** 는 한마디로 말해

> **이름(key)과 값(value)의 쌍으로 이루어진 동적 자료구조이자, 속성과 동작을 함께 담는 컨테이너**

입니다.
하지만 자바·C++ 같은 클래스 기반 언어에 익숙하면, JS의 객체는 개념적으로 꽤 다르게 느껴집니다. 그래서 정의를 단계적으로 정리해볼게요.

---

## 1. 가장 핵심적인 정의

자바스크립트의 객체는:

* **프로퍼티(property)의 집합**
* 프로퍼티 = **키(key: 문자열 또는 Symbol) → 값(value: 어떤 타입이든 가능)**

```js
const obj = {
  name: "Lee",
  age: 30,
  greet: function () {
    console.log("Hello");
  }
};
```

* `name`, `age`, `greet` → 프로퍼티
* 값은 원시값, 객체, 함수 모두 가능
* **함수도 값이 될 수 있기 때문에**, 메서드 개념이 자연스럽게 생김

---

## 2. 자바스크립트에서 객체의 본질

### 📌 “객체 = 해시맵 + 동작”

JS 객체는 본질적으로:

* **동적 해시 테이블**에 가깝고
* 실행 중에 프로퍼티를 **추가 / 삭제 / 수정** 가능

```js
obj.address = "Seoul";   // 런타임에 추가
delete obj.age;         // 런타임에 삭제
```

👉 클래스 기반 언어처럼 “필드 구조가 고정”되어 있지 않음

---

## 3. 자바스크립트 객체의 가장 중요한 특징

### ① 모든 객체는 **프로토타입을 가진다**

```js
const obj = {};
```

이 객체는 내부적으로 다음과 같은 구조를 가짐:

```
obj
 ├─ own properties
 └─ [[Prototype]] → Object.prototype → null
```

* 객체는 자기 자신에게 없는 프로퍼티를
* **프로토타입 체인**을 따라 탐색함

```js
obj.toString(); // Object.prototype.toString
```

---

### ② 함수도 객체다

```js
function Foo() {}
```

* `Foo`는 **호출 가능한 객체**
* 그래서 프로퍼티를 가질 수 있음

```js
Foo.x = 10;
```

📌 단, **모든 객체가 함수는 아니지만 모든 함수는 객체**

---

## 4. 객체 생성 방식

자바스크립트는 **클래스 없이도 객체 생성** 가능

### 1) 객체 리터럴 (가장 기본)

```js
const obj = { a: 1 };
```

### 2) 생성자 함수

```js
function Person(name) {
  this.name = name;
}
const p = new Person("Lee");
```

### 3) Object.create

```js
const proto = { greet() {} };
const obj = Object.create(proto);
```

### 4) class 문법 (문법적 설탕)

```js
class Person {
  constructor(name) {
    this.name = name;
  }
}
```

📌 `class`는 **프로토타입 기반 객체 모델 위의 문법적 추상화**일 뿐

---

## 5. 자바 관점에서 오해하기 쉬운 부분

| 자바             | 자바스크립트           |
| -------------- | ---------------- |
| 객체는 클래스의 인스턴스  | 객체는 그냥 객체        |
| 클래스가 먼저 존재     | 객체가 먼저 존재        |
| 상속은 클래스 간      | 상속은 객체 간 (프로토타입) |
| 구조가 컴파일 타임에 고정 | 구조가 런타임에 변경 가능   |

👉 JS에서는
**“객체가 중심이고 클래스는 선택사항”**

---

## 6. 한 문장으로 정리

> **자바스크립트의 객체란, 프로퍼티들의 동적 집합이며, 다른 객체를 프로토타입으로 참조함으로써 행동을 공유하는 기본 실행 단위이다.**
